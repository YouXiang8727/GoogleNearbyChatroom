# 專案相關 Skills

## 專案目標
開發一個不需要網路的多人即時聊天室。

## 技術核心
*   **Google Nearby Connections**: 使用 P2P 連結技術實作近距離通訊。

## 功能規範
1. **權限管理**:
    *   App 啟動後需引導使用者賦予必要權限（位置、藍牙、附近裝置等）。
2. **連線模式**:
    *   **搜尋 (Discovery)**: 搜尋附近已建立的聊天室。
    *   **建立 (Hosting)**: 使用者可自行建立聊天室作為 Host。
3. **即時通訊**:
    *   在連線成功後，實現多人即時訊息交換。
    *   **多媒體傳輸**: 支援圖片、影片與 GIF 傳輸。
    *   **傳輸限制**: 實作 10MB 檔案大小限制，並在超限時主動提示。
    *   **媒體預覽與播放**:
        *   影片傳輸完成後自動產生預覽圖（封面）。
        *   整合 Media3 (ExoPlayer) 實作內建影片播放器。
        *   **全螢幕檢視與下載**: 支援點擊圖片、影片或 GIF 進行全螢幕展開檢視，並在全螢幕模式下提供下載按鈕，將媒體檔案儲存至手機相簿。
    *   **成員清單側欄 (Drawer)**: 
        *   點擊 Top Bar 標題或右側選單圖示可從**右側**彈出成員清單。
        *   清單上半部顯示目前聊天室內的所有成員（使用者名稱與短 ID）。
        *   房主 (Host) 的名稱會排在清單第一位，並附帶顯眼的 "Host" 標籤。
        *   **踢人功能**: 房主可點擊成員旁的「移除」圖示將其踢出聊天室，系統會同步發送公告訊息，被踢出的成員將自動斷開連線。
    *   **訊息狀態與通知**:
        *   **傳送狀態**: 顯示「傳送中、已傳送、失敗」狀態，並精確計算多媒體檔案的所有資料包傳輸完畢才顯示「已傳送」。
    *   **背景通知與生命週期**:
        *   整合 `ProcessLifecycleOwner` 實作智慧型背景偵測，僅在 App 處於背景時發送新訊息通知。
        *   支援 Android 13+ `POST_NOTIFICATIONS` 權限，並將其設為非強制性（即便被拒絕也不會攔截核心連線功能）。
    *   **權限管理策略**:
        *   **按需請求 (At-need Request)**: 嚴格遵循 Android 現代開發規範，權限僅在該功能被使用時才觸發請求，避免啟動時強制索取非連線核心權限（如媒體存取、通知）。
        *   實作「按需請求」機制：Nearby 連線相關權限僅在開始搜尋/建立房間時檢查；照片與影片權限僅在使用者點擊「+」準備選取媒體時檢查。
        *   當核心權限被拒絕時，自動引導使用者跳轉至 Android 系統設定頁面。
    *   **儲存管理與隔離**:
        *   **資料夾隔離**: 以 `chatroomId` (Host 的 Unique ID) 為目錄名稱儲存該房間專屬的所有媒體檔案，確保不同連線組合的實體檔案隔離。
        *   **徹底清除歷史**: 支援在退出聊天室時彈出 Dialog，若選擇刪除，將同步清空 Room 紀錄與對應的實體媒體資料夾。
        *   **自動清理**: 影片檔案於 7 天後自動刪除，但保留封面預覽圖，並在 UI 上對過期媒體疊加「EXPIRED」浮水印。
    *   **訊息轉發 (Relaying)**: Host 需負責將 Client 發送的文字與媒體訊息轉發給聊天室內的其他所有成員（已排除轉發給原始發送者以避免重複）。
    *   **身份識別**: 實作基於 `ANDROID_ID` 的唯一身份系統，顯示使用者名稱時附加短 ID (Short ID)。若未設定名稱，預設使用手機型號（如 Pixel 9）。
    *   訊息需整合本地資料庫 (Room) 進行持久化儲存。
4. **狀態管理**:
    *   **系統訊息**: 成員進入或離開時，自動產生包含「使用者名稱 #短ID」的系統通知。
    *   **資源釋放**: 離開相關頁面或關閉 App 時，必須主動停止 Hosting/Discovery 並斷開所有 Nearby 連線。
5. **UI/UX 規範**:
    *   **輸入穩定性**: 聊天輸入框 (`TextField`) 應保持固定高度，不應隨換行或長文字自動擴展，以維持對話列表的可視範圍穩定（如使用 `singleLine = true`）。
6. **離線運作**:
    *   所有通訊必須在無網路環境下透過 Google Nearby API 運作。
