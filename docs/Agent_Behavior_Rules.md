# AI Agent行為規則

1. **回覆語言**: 請使用繁體中文回覆
2. **專注當前需求**: 僅處理使用者最新提出的需求，不要延續、補完或恢復先前已取消、中斷或未完成的任務。
3. **取消即終止**: 若使用者中斷、取消或切換話題，應視為舊任務已終止，除非使用者明確要求繼續。
4. **避免推測需求**: 不要根據歷史對話自行推測使用者仍想完成某個功能或重構。
5. **需求優先級**: 目前的使用者請求優先於歷史上下文、舊計畫與先前未完成的實作。
6. **最小修改原則**: 實作功能時，優先採用最小且安全的修改，避免不必要的大範圍調整。
7. **禁止無關修改**: 不要主動修改與當前需求無關的檔案、模組或功能。
8. **避免過度重構**: 不要因為小功能或小 UI 修改而重構整體架構、State Flow 或專案結構。
9. **大型調整需確認**: 若需要大規模重構、搬移 package、調整架構或新增抽象層，必須先詢問使用者。
10. **維持既有風格**: 除非使用者要求，否則應優先遵循目前專案既有的架構、命名與實作模式。
11. **避免過度抽象化**: 不要隨意新增 BaseClass、抽象層、Manager、Helper 或過度泛化的 reusable 元件。
12. **技術債處理方式**: 若發現技術債、過時設計或可優化部分，可提出建議，但不要直接全面修改。
13. **Context 使用規範**: 不要將 prototype、temporary workaround、測試程式碼或暫時方案視為正式架構規則。
14. **需求不明先詢問**: 若需求存在歧義或可能影響架構，應先詢問使用者，而非自行擴大實作範圍。
15. **Compose 修改規範**: 避免因單一畫面需求而重構整個 Compose State 架構或 Navigation Flow。
16. **Feature 邊界控制**: 完成功能時，應以目前需求範圍為界，不額外延伸功能或加入未要求的功能。
17. **規範同步**: 每次新增或修改功能後，需判斷是否需要將相關實作規範、技術核心或功能規範補充到相對應Skills.md當中，以確保文件準確反映專案現狀。
